AI Prompt Guide
Student reference for every lesson · Copy prompts · Read the code · Learn the cycle
Why are we coding with AI in plain English? — The way humans communicate with computers has changed dramatically over 70 years. Each generation of programming language made computers easier to instruct — until today, where you can describe what you want in natural language and AI translates it into code. This is the context for everything you will do in this course.
1940s – 1950s
Machine Language
Binary · 0s and 1s
Programmers wrote raw binary numbers — the only language a CPU actually understands. Writing even a simple calculation required hundreds of 1s and 0s. One wrong bit crashed everything.
💾
10110000 01100001
10001001 11000011
// Add 97 to register A
↓
MOV AL, 61h
ADD AL, BL
// Add two values
📋
1950s
Assembly Language
x86 ASM · MASM
Human-readable abbreviations replaced binary. MOV, ADD, SUB — each mapped directly to one CPU instruction. Still required knowing exactly how hardware worked. Programmers were rare specialists.
↓
1957 – 1970s
High-Level Languages
FORTRAN · COBOL · C
A compiler translated human-readable code into machine code automatically. One line of FORTRAN replaced dozens of assembly instructions. Scientists could finally write code themselves — without being hardware engineers.
🔬
DO I = 1, 10
  SUM = SUM + I
END DO
// FORTRAN loop (1957)
↓
class Dog {
  String name;
  void bark() { ... }
}
// Java object (1995)
🏗️
1980s – 1990s
Object-Oriented Languages
C++ · Java · C#
Code was organised into objects — self-contained units with data and behaviour. Large teams could build complex software (operating systems, games, business apps) because objects kept complexity manageable.
↓
1990s – 2010s
Scripting & Web Languages
Python · JavaScript · Ruby
Shorter, more readable syntax. No compilation step needed. Python reads almost like English. JavaScript runs directly in browsers — making the web interactive. Millions of new programmers entered the field. p5.js is built on JavaScript.
🌐
for name in names:
  print(f"Hello {name}")

// Python — reads like English
↓
⓪ Now · 2020s — Where You Are Today
AI / Natural Language Coding
You describe what you want in plain English — AI translates it into working code instantly. The barrier between human intention and computer execution has never been lower. But the programmer who understands the code AI generates is far more powerful than one who only pastes it. This course teaches you both: the concepts behind the code, and how to direct AI precisely.
"Draw a Dancheong pattern with repeating circles in red, blue, and green. Make it interactive so colours change on click."
The Consistent Pattern

Every generation of programming language had the same goal: make it easier for humans to tell computers what to do. Machine code → Assembly → High-level → OOP → Scripting → AI. Each step raised the abstraction level. You are learning at the highest abstraction level ever available.

Why Concepts Still Matter

AI writes in JavaScript and p5.js — which means it uses variables, loops, functions, arrays, and objects. If you don't know what these are, you can't evaluate whether the AI code is correct. The 8 concepts in this course are the vocabulary you need to read AI output, not just run it.

What is Vibe Coding? — You describe what you want in plain English, AI writes the code, you read it, run it, and modify it. You are the designer and director. AI is your assistant. Never paste code you don't understand.

Anatomy of a Good Prompt

Every strong prompt has 4 ingredients. Missing any one gives a weaker result.

🔧
Library — always start with p5.js
"Using p5.js, …"
🎨
Visual — describe exactly what it looks like
"… draw 5 circles in a cross …"
🎯
Colour — give hex codes
"… red #C0392B, blue #1A5276 …"
🖱
Interaction — say what click/key does
"… click to change colour"
After AI responds — always do this
  • Find every variable — what does it store?
  • Find every function — what does it do?
  • Find every loop — what does it repeat?
  • Find every condition — when is it true?
  • Change one number — predict before running
  • Add one comment to a line you didn't understand
Golden Rules
✅
Ask for one feature at a time
✅
Ask AI to add comments on every line
✅
Paste your existing code when asking for changes
✅
If wrong, describe what is wrong specifically
🚫
Never ask AI to write the whole project
🚫
Never submit code you can't explain line by line

The quality of your prompt determines the quality of the code. Vague prompts give vague results. Specific prompts give useful, readable, educational code.
Good vs Bad Prompt Comparison
🚫 Weak Prompt✅ Strong Prompt
"make a circle""p5.js: draw a circle at canvas centre, Dancheong red #C0392B, size 200. Comment every line."
"add colours""Change fill to Dancheong blue #1A5276 on mouse click. Do not change any other code."
"make it move""Add sin(frameCount * 0.05) so the circle pulses in size between 80 and 200."
"fix my code""Error: ReferenceError: palette is not defined line 18. Code: [paste]. What causes this?"
"write my whole project""Add ONE feature to my existing code: a click counter shown in the top-right corner."
"make it better""My BPM bar is full width. Change it to use map(song.bpm, 60, 180, 0, 300). Only that line."
Prompt Specificity Scale
⭐
Level 1 — Too vague: "make a Korean pattern" → AI guesses everything
⭐⭐
Level 2 — Basic: "p5.js Dancheong circles" → result is unpredictable
⭐⭐⭐
Level 3 — Good: library + visual + colours + interaction → useful result
⭐⭐⭐⭐
Level 4 — Expert: Level 3 + existing code + scope limit → precise, safe result

Bugs are clues, not failures. The console error tells you exactly what went wrong. Read the error → understand it → describe it to AI → apply the fix → understand the fix.
Common Error Types in p5.js
ErrorMeaningCommon Cause
ReferenceErrorVariable doesn't existTypo or used before let
TypeErrorWrong type usedMissing () on a function
SyntaxErrorCode unreadableMissing } or )
undefinedNo value assignedDeclared but never set
Blank canvasNo error, no outputnoLoop() or wrong ID
How to Use the Browser Console
1️⃣
Press F12 (Win) or Cmd+Option+J (Mac)
2️⃣
Click the Console tab — red errors appear here
3️⃣
Click the file:line link to jump to the broken line
4️⃣
Add console.log(myVar) to print values while running
5️⃣
Copy the full error → paste to AI with your code
The 3 Classic Bugs (Lesson 10)
let c = pallete[i % palette.length]; // 🐛 Bug 1: typo ellipse(0, 0, r*2); // 🐛 Bug 2: missing arg pop; // 🐛 Bug 3: not a call
🔧
Bug 1: pallete → palette — typo (ReferenceError)
🔧
Bug 2: ellipse(0,0,r*2) → ellipse(0,0,r*2,r*2) — 4 args needed
🔧
Bug 3: pop → pop() — functions need parentheses (TypeError)

Lesson 11 — Free Project. Plan on paper first. Build the core feature before asking AI for anything. Use AI for one specific stuck moment at a time. Explain every line at Demo Day.
Project Planning Checklist

Answer these before opening the code editor:

  • What does it look like? Sketch on paper (2 min)
  • Korean cultural element? Dancheong / Hangeul / K-POP / Obangsaek
  • What data do I need? Array? Object? What properties?
  • What interaction? Mouse click / key press / hover
  • What function will I write? Name it + list parameters
  • Which concept am I unsure of? Start there.
30-Minute Build Sprint
📝
0–5 min: Write the plan. What is the ONE core feature?
⌨️
5–20 min: Build the core feature only. No polish yet.
🤖
20–25 min: Use AI for ONE specific stuck thing. Read all output.
✨
25–30 min: Polish. Colours, layout, interaction. Add comments.

Minimum viable = one screen + one interaction + one Korean cultural element + comments on every function.
